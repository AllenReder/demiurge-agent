# 06:59:07 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 prompt interaction 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema。

## Hermes 对比

- Hermes 的 clarify gateway primitive 把 register、wait、resolve、session cleanup 放在独立 module，平台 adapter 只负责渲染按钮或文本 fallback，再把用户选择回传给 shared primitive。
- 当前 Demiurge 已有 `UserPromptRequest` 和 interaction route，但 choice 规则仍散在 adapter：
  - TUI 用 `_normalize_prompt_answer` 把数字 reply 映射为 choice。
  - Text channel 用 `_pending_choices`、`_consume_pending_choice` 和 `_prompt_text`。
  - Telegram 复制 `_pending_choices`、数字 reply 映射、callback index 解析、prompt text 和 button rows。
- 这还不是完整 Hermes-style blocking clarify store，但可以先把纯 in-process choice lifecycle 收拢，减少后续 approval/prompt lifecycle 重构时的 adapter 知识泄漏。

## 设计

新增 `demiurge.runtime.prompts`：

- `PromptChoiceRuntime`：按 `conversation_key` 保存 pending choices，消费 text reply 或 callback index。
- 无 choices 的 prompt 会清理同一 `conversation_key` 的旧 pending choices，避免 adapter 时代的陈旧状态泄漏。
- `PromptChoiceResolution`：返回 `text`、`consumed`、`matched_choice`、`index`，让 adapter 不需要知道 choice 解析细节。
- `format_prompt_text(question, choices)`：统一文本 fallback。
- `normalize_prompt_answer(answer, choices, empty="raw|first")`：供 TUI prompt 和 session resume prompt 使用。
- `choice_callback_data(index)` 和 `choice_button_rows(choices, label_limit=32)`：供 Telegram buttons 使用。

Adapter 接线：

- `TextChannelBridgeBase` 用 `PromptChoiceRuntime` 替换 dict，并删除 `_consume_pending_choice` / `_prompt_text`。
- `TelegramInteractionBridge` 用 `PromptChoiceRuntime` 替换 dict，并删除 duplicated choice parsing / prompt text / reply markup helpers。
- `TuiInteractionBridge.reply_prompt` 用 shared `normalize_prompt_answer(..., empty="first")`，保留 TUI 自己的 prompt id/future lifecycle。

## 优势

- locality：choice 解析 bug 集中在一个 module。
- leverage：TUI、text channels、Telegram 同时复用。
- depth：adapter interface 缩小为 “remember prompt choices / consume inbound / render transport-specific output”。
- testability：新增 `tests/runtime/test_prompts.py` 直接覆盖 shared interface，现有 channel tests 继续覆盖 wiring。

## Implementation steps

1. 新增 `demiurge/runtime/prompts.py` 和 `tests/runtime/test_prompts.py`。
2. 更新 `demiurge/channels/base.py` 使用 `PromptChoiceRuntime` 和 `format_prompt_text`。
3. 更新 `demiurge/channels/telegram/bridge.py` 使用 `PromptChoiceRuntime`、`format_prompt_text`、`choice_button_rows`。
4. 更新 `demiurge/ui_gateway/bridge.py` 使用 `normalize_prompt_answer`，删除本地 choice normalize helper。
5. 运行验证：
   - `uv run pytest tests/runtime/test_prompts.py tests/channels/telegram/test_channels.py tests/channels/test_ui_gateway.py -q`
   - `uv run python -m compileall demiurge/runtime/prompts.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py demiurge/ui_gateway/bridge.py tests/runtime/test_prompts.py tests/channels/telegram/test_channels.py`
   - `git diff --check -- demiurge/runtime/prompts.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py demiurge/ui_gateway/bridge.py tests/runtime/test_prompts.py tests/channels/telegram/test_channels.py goal/06-59-07.md`

## Non-goals

- 不实现完整 blocking clarify store。
- 不改变 approval request/decision lifecycle。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
