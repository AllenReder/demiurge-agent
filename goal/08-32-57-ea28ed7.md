# 08:32:57 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 text outbound route 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema。

## Hermes 对比

- Hermes 的 gateway/platform base 中有 `_reply_anchor_for_event()` 和 `_thread_metadata_for_source()`：route metadata 是平台发送前的 shared interface，不在每个 delivery/tool 分支里重复解析。
- Hermes 的 adapter `send()` 仍保留平台具体实现，例如 Telegram thread、reply、chunk、rich message 等；shared route module 不吞掉平台 adapter 的职责。
- 当前 Demiurge 已有 `TextOutboundDeliveryRuntime` 负责 outbound item ordering，但 route metadata 仍泄漏在 adapter：
  - `TextChannelBridgeBase._deliver_delivery()`、`_deliver_tool_results()`、`_deliver_tool_call()` 都重复读取 `outbound.metadata["source"]` 和 `reply_to`。
  - `TelegramInteractionBridge._deliver_text()`、`_deliver_tool_results()`、`_deliver_tool_call()`、`_deliver_delivery()` 也重复相同 target parsing。
  - Text 和 Telegram 都有 media block fallback text 生成逻辑。

## 设计

扩展 `demiurge.runtime.outbound_delivery`：

- 新增 `TextOutboundTarget`：
  - `source: str`
  - `reply_to: str | None`
  - `metadata: dict[str, object]`
- 新增 `text_outbound_target(outbound)`：
  - 没有 `source` 时返回 `None`，保持当前 fail-closed/no-send 行为。
  - 统一把 `source` / `reply_to` 转成 string，并复制 metadata。
- 新增 `media_block_fallback(block)`：
  - 集中 artifact fallback 文本生成。
- 新增 `delivery_text_chunks(delivery)`：
  - 集中普通 text delivery 和 text fallback chunks。
  - Text adapter 直接使用；Telegram adapter 继续保留 native media delivery，只复用 fallback renderer。

Adapter 接线：

- `TextChannelBridgeBase` 用 `text_outbound_target()` 和 `delivery_text_chunks()` 删除本地 route/fallback 方法。
- `TelegramInteractionBridge` 用 `text_outbound_target()` 删除重复 target parsing，用 `media_block_fallback()` 删除本地 fallback 方法。
- 保留 Telegram `_send_text()`、rich message、reply_to_mode、media upload、tool message edit 逻辑在 adapter 内。

## 优势

- locality：outbound route 解析 bug 集中在一个 runtime module。
- leverage：Text 和 Telegram adapter 共用 route interface。
- depth：adapter delivery method 不再知道 metadata shape，只发送 prepared target。
- testability：runtime tests 直接覆盖 target extraction 和 fallback rendering，channel tests 继续覆盖 Telegram-specific send behavior。

## Implementation steps

1. 扩展 `demiurge/runtime/outbound_delivery.py`：添加 `TextOutboundTarget`、`text_outbound_target()`、`media_block_fallback()`、`delivery_text_chunks()`。
2. 扩展 `tests/runtime/test_outbound_delivery.py` 覆盖 target extraction、missing source、fallback chunks。
3. 更新 `demiurge/channels/base.py` 使用 runtime outbound helpers，删除本地 `_delivery_text_chunks()` 和 `_media_block_fallback()`。
4. 更新 `demiurge/channels/telegram/bridge.py` 使用 `text_outbound_target()` 和 `media_block_fallback()`。
5. 运行验证：
   - `uv run pytest tests/runtime/test_outbound_delivery.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py -q`
   - `uv run python -m compileall demiurge/runtime/outbound_delivery.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/runtime/test_outbound_delivery.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py`
   - `git diff --check -- demiurge/runtime/outbound_delivery.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/runtime/test_outbound_delivery.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py goal/08-32-57.md`

## Non-goals

- 不实现 native media delivery runtime。
- 不改变 Telegram `reply_to_mode`、rich message、tool edit、media upload 行为。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
