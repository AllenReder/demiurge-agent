## Reference Comparison

### Hermes reference

Hermes keeps gateway delivery concerns in named Modules:

- `gateway/delivery.py` owns delivery target parsing, fallback behavior, and platform routing decisions.
- `gateway/platforms/base.py` owns platform adapter send semantics and media-specific behavior.
- `agent/turn_finalizer.py` demonstrates the same lifecycle lesson: side-effect-heavy post-loop work is easier to reason about when it has a named Module and explicit inputs.

The transferable lesson is that delivery is a host-owned lifecycle product, not incidental runner glue.

### Demiurge current state

This branch now has:

- `TurnEngine` for model/tool loop orchestration.
- `SlotPipelineRuntime` for input/output phase scheduling.
- `SlotContextRuntime` for authored `ctx.*` construction.
- `TurnIO` for model-step transcript and interaction-item assembly.

The remaining friction in this area is that `TurnIO` still calls back into runner private delivery normalization:

- `_apply_delivery_request(...)`
- `_delivery_artifact_dict(...)`
- `_apply_deliver_effect(...)`
- `_payload_text(...)`
- `_delivery_history_content(...)`

This code validates delivery policy, stores artifacts, writes assistant history, appends durable outbox work for transient deliveries, emits runtime events, and constructs visible `InteractionDelivery` values.

Deletion test: deleting this runner block would force the same artifact/history/outbox/fallback logic to reappear in `TurnIO`, slot context delivery helpers, and legacy effect handling. It is a deep Module trapped in the runner.

## This Loop Scope

Extract a `ModuleDeliveryRuntime` Module that owns authored module delivery request normalization and legacy deliver-effect conversion.

This is behavior-preserving. It changes internal Module placement and host Adapter shape only; it does not change Agent Core layout, Agent Slot manifests, Package recipes, authored SDK methods, outbox dispatch behavior, or channel behavior.

## Planned Code Changes

- Add `demiurge/runtime/module_delivery.py`.
- Move the following implementation from runner into that Module:
  - applying a `DeliveryRequest`;
  - validating history policy, delivery mode, kind, and target;
  - normalizing content blocks;
  - storing artifacts and building history/delivery artifact dicts;
  - writing assistant history through `SessionRuntime.append_delivery_message`;
  - enqueuing transient `delivery.queued` and `work.enqueued` runtime events;
  - emitting `message.persisted`, `delivery.completed`, and `delivery.degraded`;
  - converting legacy `EffectRequest` deliver effects into `DeliveryRequest`.
- Add a `ModuleDeliveryHost` Protocol and `RunnerModuleDeliveryHost` Adapter.
- Instantiate `self.module_delivery = ModuleDeliveryRuntime(RunnerModuleDeliveryHost(self))` in runner.
- Update runner:
  - `_apply_delivery_request(...)` delegates to `self.module_delivery.apply_request(...)`;
  - `_apply_deliver_effect(...)` delegates to `self.module_delivery.apply_deliver_effect(...)`;
  - remove runner-local delivery helper methods that become unused.
- Keep these concerns host-owned for this loop:
  - delivery dispatch and routing through `DeliveryRuntime`;
  - background task tracking;
  - tool execution and effect handling for non-delivery effect types;
  - child-agent run/spawn/delegation.
- Add focused tests in `tests/runtime/test_module_delivery.py`:
  - persisted text delivery writes message, delivery outbox work, and visible `InteractionDelivery`;
  - transient delivery writes outbox/runtime events without assistant history;
  - non-text visible delivery requires `history_text` when history is writable;
  - artifact delivery stores artifacts and emits degradation for TUI text fallback;
  - invalid history policy/mode/kind/target raise;
  - legacy deliver effect delegates through the same request path.

- `SessionTurnStepRunner` loses authored delivery normalization responsibility.
- `TurnIOHost` and future child-agent runtime no longer need an unnamed runner-private delivery seam.
- Delivery policy, artifact persistence, history writes, and outbox enqueue semantics become directly testable through one Module Interface.
- Future bridge/channel/TUI changes have a narrower, named host-owned Module to work against.
