## Reference Comparison

### Hermes reference

Hermes keeps gateway turn lifecycle rules in shared gateway modules:

- `gateway/session.py` names `SessionSource` and `build_session_key(...)`, so platform adapters share the same conversation identity model.
- `gateway/platforms/base.py` owns active-session guards, pending-message state, interruption, cancellation, and pending-drain handoff for all platform adapters.
- Hermes tests such as `tests/gateway/test_pending_drain_race.py` and `tests/gateway/test_command_bypass_active_session.py` treat active-session state and pending-drain ordering as gateway-level invariants, not Telegram-only behaviour.

The transferable lesson is not Hermes's exact `Event`/pending-message structure. Demiurge should name the shared turn lifecycle module so text channels and Telegram stop duplicating running/queued/interrupted/drain behaviour.

### Demiurge current state

This branch now has:

- `TurnEngine` for model/tool loop orchestration.
- `SlotPipelineRuntime` for input/output phase scheduling.
- `SlotContextRuntime` for authored `ctx.*` construction.
- `TurnIO` for model-step transcript and interaction-item assembly.
- `ModuleDeliveryRuntime` for authored delivery request application.
- `ChildAgentRuntime` for child-agent lifecycle.
- `CompletionInbox` for background completion claim, inbound construction, merge metadata, and ack.
- `InboundQueueRuntime` for inbound queue selection, completion merge, and completion-preserving clear.
- `ConversationIngressState` for route memory, session lookup, completion route construction, and completion claim.

The remaining shallow cluster is conversation turn lifecycle:

- `TextChannelBridgeBase._handle_busy_inbound(...)` and `TelegramInteractionBridge._handle_busy_inbound(...)` duplicate the same queue/interrupt decision.
- `_start_turn(...)`, `_drain_next_queued_input(...)`, `_clear_queue(...)`, `_next_queued_input(...)`, and `_enqueue_task_completion(...)` repeat the same active-task and queue-drain rules.
- `_cancel_active(...)` is mostly shared, with Telegram adding one platform-specific pre-cancel hook for pending approvals.

Deletion test: deleting either adapter's lifecycle helpers would force the same active-task and pending-drain logic to reappear. Two adapters justify a real seam.

## This Loop Scope

Extract a `ConversationTurnController` module that owns shared conversation turn lifecycle rules for a single `ConversationIngressState`.

This is behaviour-preserving. It changes internal module placement and adapter call sites only; it does not change Agent Core layout, Agent Slot manifests, Package recipes, authored SDK methods, slash command semantics, approval semantics, busy mode meanings, delivery rendering, task completion schema, or Telegram polling behaviour.

## Planned Code Changes

- Extend `demiurge/runtime/ingress.py` with:
  - `BusyInboundDecision`;
  - `ConversationTurnController`.
- Move shared lifecycle behaviour into `ConversationTurnController`:
  - `running`;
  - `handle_busy_inbound(inbound, notify=...)`;
  - `start(inbound, run)`;
  - `finish(task)`;
  - `drain_next(run)`;
  - `cancel_active(before_cancel=..., timeout_seconds=...)`;
  - `clear_queue(preserve_completions=...)`;
  - `next_queued_input()`;
  - `queue_and_drain_if_idle(inbound, run)`;
  - `enqueue_completion(inbound, run)`.
- Update `TextChannelBridgeBase`:
  - delegate busy queue/interrupt state transitions to the controller while keeping text formatting local;
  - delegate start/finish/drain/cancel/clear/next/completion enqueue to the controller.
- Update `TelegramInteractionBridge`:
  - delegate busy queue/interrupt state transitions to the controller while keeping Telegram send formatting local;
  - delegate start/finish/drain/clear/next/completion enqueue to the controller;
  - keep Telegram-specific pending approval cancellation as the controller's `before_cancel` hook for `/stop`.
- Add focused tests in `tests/runtime/test_ingress.py`:
  - queue busy mode queues without cancelling active work;
  - interrupt busy mode notifies before cancelling active work;
  - `finish(...)` clears only the active task it owns;
  - completion enqueue queues while active and starts immediately while idle;
  - `queue_and_drain_if_idle(...)` starts queued input when idle.

## Out Of Scope

- Moving adapter-specific typing indicators.
- Moving adapter-specific error rendering.
- Moving slash command dispatch into a shared command runtime.
- Moving pending approval state into runtime.
- Changing queue vs interrupt behaviour.
- Changing route memory or completion claim schema.
- Changing Agent Core / Agent Slot / Package authored surface.
- Copying Hermes implementation code.

## Expected Benefits

- The conversation turn lifecycle becomes one deep module interface instead of two copied adapter helper clusters.
- Locality improves: active-task state, cancellation, queue drain, and completion enqueue rules live together.
- Leverage improves: Telegram and all `TextChannelBridgeBase` adapters reuse the same lifecycle module.
- Tests can target the lifecycle interface directly, matching the codebase-design principle that the interface is the test surface.
