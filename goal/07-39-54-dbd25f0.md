# 07:39:54 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 background completion delivery 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema，不改变 durable task/control-plane state machine。

## Hermes 对比

- Hermes/OpenClaw 的后台任务语义强调 session routing、delivery ownership、run history 和 cleanup，而不是把 completion 投递规则散在平台 adapter。
- 当前 Demiurge 已有 durable completion source of truth：`RuntimeTaskWorker` / `CompletionInbox` / `ConversationTurnController` 负责 claim、merge、enqueue 和 ack 的核心逻辑。
- 但 adapter-facing glue 仍重复：
  - Text channel 在 `handle_inbound()` 中判断 `is_background_completion()`，再 `_merge_stored_task_completions()`。
  - Telegram channel 复制同样的 merge 路径和 enqueue path。
  - TUI 用 `_merge_pending_completions_into()` 和 `_enqueue_task_completion()` 走同一控制器，但 owner id、fallback source 和 status emit 仍由 adapter 自己拼。
- 这使 adapter interface 仍然浅：每个 adapter 都知道 completion claim owner id、background completion skip 规则、stored completion merge 和 task-worker callback enqueue。

## 设计

新增 `demiurge.runtime.completion_delivery`：

- `CompletionDeliveryRuntime`：
  - 初始化参数：`channel`、`merge_owner_id`、`enqueue_owner_id`、可选 `task_worker`、`require_source`、`fallback_source`。
  - `merge_pending_into(state, inbound, fallback_source=None)`：若 inbound 已是 background completion，直接返回；否则统一 claim stored completions 并 merge 到用户 turn。
  - `enqueue_event(state, event, run, before_enqueue=None)`：统一 claim completion event、queue/start turn，并返回 `CompletionEnqueueResult`。
- 该 module 是 adapter-facing runtime seam；它不拥有 durable task state，不替代 `CompletionInbox` 或 `ConversationTurnController`。

Adapter 接线：

- `TextChannelBridgeBase` 删除 `_merge_stored_task_completions()` 的 controller glue，改用 `CompletionDeliveryRuntime(...).merge_pending_into()` / `.enqueue_event()`。
- `TelegramInteractionBridge` 同样改用 shared runtime。
- `TuiInteractionBridge` 用 shared runtime 替代 `_merge_pending_completions_into()` 的 controller glue，并保留 TUI-only status notice/emit 行为。

## 优势

- locality：completion delivery 的 adapter-facing policy 集中在一个 module。
- leverage：Text、Telegram、TUI 三个 adapter 同时复用。
- depth：adapter interface 缩小为 “找到 conversation state / 提供 run callback / 可选显示 notice”。
- testability：新增 `tests/runtime/test_completion_delivery.py` 覆盖 merge skip、stored completion merge 和 enqueue event interface；现有 TUI tests 继续覆盖 user-visible behavior。

## Implementation steps

1. 新增 `demiurge/runtime/completion_delivery.py` 和 `tests/runtime/test_completion_delivery.py`。
2. 更新 `demiurge/channels/base.py` 使用 `CompletionDeliveryRuntime`。
3. 更新 `demiurge/channels/telegram/bridge.py` 使用 `CompletionDeliveryRuntime`。
4. 更新 `demiurge/ui_gateway/bridge.py` 使用 `CompletionDeliveryRuntime`。
5. 运行验证：
   - `uv run pytest tests/runtime/test_completion_delivery.py tests/channels/test_ui_gateway.py tests/channels/telegram/test_channels.py tests/channels/test_commands.py -q`
   - `uv run python -m compileall demiurge/runtime/completion_delivery.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py demiurge/ui_gateway/bridge.py tests/runtime/test_completion_delivery.py`
   - `git diff --check -- demiurge/runtime/completion_delivery.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py demiurge/ui_gateway/bridge.py tests/runtime/test_completion_delivery.py goal/07-39-54.md`

## Non-goals

- 不改变 `RuntimeTaskWorker.wait(..., consume_completion=True)` 的 duplicate-reply prevention seam。
- 不改变 durable runtime work item / ack state machine。
- 不改变 Telegram/TUI/text channel 的 user-visible delivery copy。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
