## Reference Comparison

### Hermes reference

Hermes's tool presentation path differs from Demiurge:

- Hermes terminal tools return JSON-like strings with fields such as `output`, `exit_code`, and `error`.
- Hermes TUI/gateway presentation code summarizes parsed tool output rather than relying on a `display_output` field.
- Platform delivery code still owns message sending, chunking, and adapter-specific lifecycle behavior.

The transferable lesson is not Hermes's exact terminal result shape. Demiurge should centralize the read-model that turns `ToolInteractionRecord` / `ToolExecutionRecord` into human-facing summaries while leaving platform delivery and edit lifecycle in adapters.

### Demiurge current state

This branch now has:

- `demiurge/slash.py` as the command catalog read-model.
- `demiurge/runtime/session_commands.py` as the session list/resume read-model.
- `demiurge/runtime/status_commands.py` as the runtime status read-model.

The remaining shallow tool display cluster is:

- `TextChannelBridgeBase._tool_call_text(...)`, `_tool_results_text(...)`, and `_tool_call_start_summary(...)`.
- `TelegramInteractionBridge._tool_call_text(...)`, `_tool_results_text(...)`, and `_tool_call_start_summary(...)`.
- `TuiInteractionBridge._tool_call_record_dict(...)`, `_historical_tool_item(...)`, and `_tool_call_start_summary(...)`.

All three adapters repeat the same read-model semantics:

- terminal start summaries use `$ <command>`;
- file tools summarize the target path;
- generic calls summarize JSON arguments;
- result text falls back as `display_output -> content`;
- summary mode shortens text;
- full mode includes arguments, result, and sometimes model output.

Deletion test: deleting a shared tool display read-model module would force the same summary/full/model-output rules back into Text, Telegram, and TUI adapters.

## Out Of Scope

- Moving Telegram tool lifecycle edit/send behavior.
- Changing TUI `interaction.deliver` protocol keys.
- Changing tool lifecycle event emission in the runner.
- Changing `ToolResult` or `ToolExecutionRecord`.
- Changing tool approval or quiet/full/summary policy resolution.
- Changing Agent Core / Agent Slot / Package authored surface.
- Copying Hermes implementation code.

## Expected Benefits

- Tool display semantics become one deeper module interface.
- Locality improves: summary/full/model-output logic is maintained in one place.
- Leverage improves: Text, Telegram, TUI live payloads, and TUI historical reconstruction reuse the same read-model.
- Tests target the display read-model interface directly while adapter tests stay focused on delivery and ordering.
