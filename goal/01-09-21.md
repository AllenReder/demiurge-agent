## Reference Comparison

### Hermes reference

Hermes does not have Demiurge's Agent Slot model, so this loop should not copy Hermes slot behavior. The relevant lesson is turn lifecycle decomposition:

- `agent/turn_context.py` names the turn prologue and returns values consumed by the loop.
- `agent/conversation_loop.py` keeps the model/tool loop focused after prologue extraction.
- `agent/turn_finalizer.py` names post-loop cleanup and result assembly.

The transferable design point is that once-per-phase runtime lifecycle behavior should be a named Module with a small Interface.

### Demiurge current state

- `demiurge/runtime/slots.py` currently contains `SlotRuntime`, a deep Module for invoking one authored slot handler.
- The actual input/output phase behavior still lives in `SessionTurnStepRunner`:
  - resolve serial and parallel slot lists
  - schedule parallel slots as background tasks
  - keep serial slot items in the turn result
  - flush parallel slot items through delivery runtime
  - assemble input `ContextContribution` values
  - assemble user text and persisted user text
  - enforce "input pipeline produced a user message"
  - fork output result clients for background slots

This means Agent Slot is a strong concept, but slot phase execution is still a shallow runner-owned implementation.

## This Loop Scope

Implement the second deepening candidate:

> Extract input/output slot phase scheduling and phase result assembly into `SlotPipelineRuntime`.

This loop intentionally does not yet move every SDK client or context construction detail out of the runner. That is the next deepening step. This loop creates the phase seam and tests it directly.

## Planned Code Changes

- Move `ModuleInputBuilder` from `runner.py` into `demiurge/runtime/slots.py`, because input phase assembly belongs with slot pipeline execution.
- Add `SlotPipelineHost` Protocol and `RunnerSlotPipelineHost` Adapter.
- Add `InputPipelineRequest`, `InputPipelineResult`, and `OutputPipelineRequest`.
- Add `SlotPipelineRuntime.run_input(...)` and `SlotPipelineRuntime.run_output(...)`.
- Change `SessionTurnStepRunner._run_input_slots(...)` and `_run_output_slots(...)` into compatibility wrappers that resolve slot lists and delegate to `SlotPipelineRuntime`.
- Add runner turn/slot-oriented methods used by the adapter:
  - emit slot event
  - track background slot task
  - run one input slot
  - run one output slot
  - flush background slot items
- Remove runner's now-obsolete `_run_async_input_slot(...)` and `_run_async_output_slot(...)`.
- Add focused tests for `SlotPipelineRuntime` using a fake host.

## Out Of Scope

- Moving bootstrap execution into `SlotPipelineRuntime`.
- Moving `InputContext` / `OutputContext` construction out of `SessionTurnStepRunner`.
- Changing Agent Core file layout, slot YAML, pipelines YAML, or package recipes.
- Rewriting channel/TUI delivery.
- Copying Hermes implementation code.

## Expected Benefits

- Better locality for serial/parallel slot phase behavior.
- A direct test surface for input/output phase orchestration.
- A stable seam for the next loop to move SDK context construction.
- Less `SessionTurnStepRunner` responsibility without changing user-visible behavior.
