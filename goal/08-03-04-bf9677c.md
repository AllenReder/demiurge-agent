# 08:03:04 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 channel runtime factory 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema，不改变 session/turn durable state machine。

## Hermes 对比

- Hermes 的 gateway/session 文档把 session source、session store、gateway runner wiring 拆成一等 runtime concern；平台 connector/adapter 负责 transport 和 normalization，不应该重复知道 runner 构造细节。
- 当前 Demiurge 已有 `InteractionRuntime`、`SessionTurnStepRunner`、`runtime_factory_for_app()`，但 host-to-channel runtime wiring 仍是浅 interface：
  - `demiurge/channels/base.py` 暴露 `runtime_factory_for_app()`，并直接知道 `SessionTurnStepRunner` 的完整构造参数。
  - `demiurge/channels/telegram/bridge.py` 复制一份 `_runtime_factory_for_app()`，也直接知道同一组 runner/app fields。
  - Matrix/Slack/Mattermost/Email/Webhook 从 `channels.base` 导入 factory，使 base module 同时承担 text adapter base class、router、env helper、runtime factory 多个角色。
- 这让 channel adapter 仍越过 seam：它们知道 host runner construction，而不是只拿到 `Callable[[conversation_key], InteractionRuntime]`。

## 设计

新增 `demiurge.runtime.interaction_factory`：

- `runtime_factory_for_app(app)`：
  - 对 minimal test app：复用 existing `InteractionRuntime(app.runner)` 行为。
  - 对 full `DemiurgeApp`：集中创建 per-conversation `SessionTurnStepRunner` 并包装为 `InteractionRuntime`。
- 该 module 是 host-owned runtime factory seam；它不拥有 channel transport，不替代 `InteractionRuntime` 或 `SessionTurnStepRunner`。

Adapter 接线：

- `demiurge/channels/base.py` 删除 runtime factory implementation，只保留 text bridge base/router/env helper。
- Telegram builder 和 text-channel builders 统一从 `demiurge.runtime.interaction_factory` 导入 `runtime_factory_for_app`。
- `demiurge/channels/registry.py` 不再从 `channels.base` re-export runtime factory。

## 优势

- locality：runner construction 变化集中在一个 runtime module。
- leverage：Telegram 和所有 text-like channels 同时复用同一个 factory。
- depth：channel adapter interface 缩小为 “接受 runtime/runtime_factory”，不再知道 runner internals。
- testability：新增 runtime-level tests 覆盖 minimal app reuse 与 full app per-conversation runner creation；existing channel registry tests 覆盖 channel builder wiring。

## Implementation steps

1. 新增 `demiurge/runtime/interaction_factory.py` 和 `tests/runtime/test_interaction_factory.py`。
2. 从 `demiurge/channels/base.py` 移除 `runtime_factory_for_app()` 和 `SessionTurnStepRunner` import。
3. 更新 Matrix/Slack/Mattermost/Email/Webhook channel imports 到 runtime module。
4. 更新 Telegram builder 删除私有 `_runtime_factory_for_app()`，统一使用 runtime module。
5. 更新 registry export，避免继续把 runtime factory 伪装成 channel registry concern。
6. 运行验证：
   - `uv run pytest tests/runtime/test_interaction_factory.py tests/channels/test_registry.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py -q`
   - `uv run python -m compileall demiurge/runtime/interaction_factory.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py demiurge/channels/matrix/bridge.py demiurge/channels/slack/bridge.py demiurge/channels/mattermost/bridge.py demiurge/channels/email/bridge.py demiurge/channels/webhook/bridge.py tests/runtime/test_interaction_factory.py`

## Non-goals

- 不改变 channel config schema。
- 不改变 runtime/session routing semantics。
- 不改变 Telegram/text-like channel user-visible behavior。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
