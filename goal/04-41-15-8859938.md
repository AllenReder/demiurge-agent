## Reference Comparison

### Hermes reference

Hermes centralizes slash command definitions and derived command views:

- `hermes_cli/commands.py` owns `COMMAND_REGISTRY`, aliases, surface flags, and lookup helpers.
- `gateway/slash_commands.py` implements gateway command behavior while consuming the same command vocabulary.
- `tui_gateway/slash_worker.py` isolates TUI slash command execution from the terminal UI process.
- `gateway/platforms/base.py` keeps platform delivery and active-session mechanics out of the command catalog.

The transferable lesson is not Hermes's exact command set or persistent worker. Demiurge should keep adapter-specific transport/rendering in adapters, but move host read-model semantics that every adapter repeats into one module interface.

### Demiurge current state

This branch now has:

- `ConversationIngressState` / `ConversationTurnController` for human-facing ingress lifecycle across Text, Telegram, and TUI.
- `ChannelCommandRuntime` for external-channel slash command classification and dispatch.
- `ScheduleFireRuntime` for one claimed schedule execution.

The remaining shallow command cluster is session list/resume read-model logic:

- `TuiInteractionBridge._sessions(...)`, `_resume(...)`, `_resolve_resume_arg(...)`, `_session_record_dict(...)`, and `_format_sessions(...)`.
- `TextChannelBridgeBase._command_sessions(...)`, `_command_resume(...)`, and `_sessions_text(...)`.
- `TelegramInteractionBridge._command_sessions(...)`, `_command_resume(...)`, and `_sessions_text(...)`.

All three adapters repeat the same host reads and choice rules:

- load recent sessions for `runner.core_id`;
- mark the active `runner.session_id`;
- render empty and non-empty session lists;
- resolve numeric choices to session ids;
- reject out-of-range numeric choices;
- pass raw session ids through.

Deletion test: deleting a shared session command read-model module after extraction would force the same session rows, active marker, and resume choice logic back into every adapter.

## Out Of Scope

- Full command catalog redesign.
- Moving `/status`, `/tools`, `/skills`, `/skill`, `/subagents`, `/new`, or `/busy`.
- Moving `runner.resume_session(...)` into the read-model module.
- Moving route binding, TUI history snapshot emission, or platform delivery into the read-model module.
- Changing session storage or persisted history schemas.
- Changing Agent Core / Agent Slot / Package authored surface.
- Copying Hermes implementation code.

## Expected Benefits

- Session command read semantics become one deeper module interface.
- Locality improves: active markers, list rendering, and choice resolution live in one place.
- Leverage improves: TUI, Telegram, and every text-channel adapter reuse the same read-model.
- Tests target the read-model interface directly instead of adapter-private helper copies.
