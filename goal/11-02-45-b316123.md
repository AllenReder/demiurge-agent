# 11-02-45 Session Compaction Runtime Plan

## Hermes vs Demiurge comparison

- Hermes 的 `ContextEngine` 明确把 compaction 当作 session/context runtime lifecycle：决定是否压缩、执行压缩、维护 token/context 状态、处理 manual `/compress` 都不是 agent facade 的职责。
- 当前 Demiurge 的 manual compaction 仍在 `SessionTurnStepRunner.compact_session(...)`：
  - runner 选择可 compact 的历史范围；
  - runner 格式化 transcript；
  - runner 构造 summary provider request；
  - runner 持久化 compaction summary；
  - runner emit `session.compaction.*` 事件并刷新 history。
- 这让 context lifecycle 继续散在 runner，而不是跟 prompt context/session history runtime 放在同一个可演化 seam。

## Breaking redesign target

新增 `SessionCompactionRuntime`：

- `SessionCompactionRuntime.compact(...)` 拥有 manual session compaction lifecycle。
- `PromptContextRuntime` 继续读取 latest compaction summary，不负责生成 summary。
- `SessionRuntime` 继续是 messages/session persistence owner。
- `SessionTurnStepRunner.compact_session(...)` 退化为 facade。

Agent Core、Agent Slot、Package 概念不变。本轮只改变 session context lifecycle ownership。

## Deep module check

- Interface size: runner 只调用 `session_compaction.compact(focus, protect_last_n)`。
- Deletion test: 删除该 module 会把 history window selection、transcript formatting、summary prompt construction、provider call、summary persistence、event emission、history refresh重新散回 runner。
- Hidden behavior: module 有真实行为。它决定哪些 turns 被压缩、如何保留最新 turns、summary wrapper marker、失败/跳过/成功状态。
- Adapter discipline: gateway `/compact` 仍只调用 runner facade；TUI/channel 不知道 transcript formatting 或 provider request shape。

## Old interfaces and runner-local behavior to delete

- 删除 runner `CompactionResult`。
- 删除 runner `SUMMARY_PREFIX` / `SUMMARY_END_MARKER` ownership。
- 删除 runner `_format_compaction_message(...)`。
- 将 runner `compact_session(...)` 改为委托。

## New seam placement

- New module: `demiurge/runtime/session_compaction.py`
  - `CompactionResult`
  - `SessionCompactionRuntime`
  - private `_format_compaction_message(...)`
  - summary marker constants
- Runner composition:
  - runner 持有 `self.session_compaction = SessionCompactionRuntime(...)`。
  - runtime 接收 session id getter、core loader/model resolver callback、provider completion callback、session runtime、event emitter、history refresh callback。

## Why this is framework redesign, not ordinary cleanup

- 这是 context/session lifecycle 的 ownership 修正。Prompt context 已经从 runner 拆出；compaction 是同一生命周期的写入端。
- 后续可以在这个 seam 上接入 Hermes-style pluggable context engine、automatic compaction、token budget accounting 和 provider-specific summary policy。
- runner 继续向 composition facade 收缩，而不是拥有 context lifecycle。

## Implementation steps

1. 新增 `demiurge/runtime/session_compaction.py` 并迁移 compaction behavior。
2. 在 runner 初始化 `self.session_compaction`。
3. 将 runner `compact_session(...)` 改为委托。
4. 删除 runner 内的 compaction constants/result/formatter。
5. 增加 focused tests 覆盖 skipped、successful summary persistence、provider empty summary error。
