## Reference Comparison

### Hermes reference

Hermes documents session identity, reset/resume behavior, queue recovery, and restart recovery as gateway/session lifecycle rules below platform adapters. Platform adapters provide message source and delivery details; session lifecycle logic is not copied into every adapter.

The transferable design is not Hermes's session store. Demiurge should keep `SessionTurnStepRunner`, `SessionRouteBinding`, and `InteractionRuntime`, but the rule that creates or resumes a session and binds the route token should live in the runtime session command module.

### Demiurge current state

This branch has already extracted runtime modules for command catalogs, status, tool display, text formatting, event commands, history display, outbound delivery planning, and completion intake.

The remaining shallow session switch cluster is in:

- `demiurge/ui_gateway/bridge.py`;
- `demiurge/channels/base.py`;
- `demiurge/channels/telegram/bridge.py`;
- `demiurge/runtime/session_commands.py`.

The adapters still repeat the same rules:

- prepare the live core before `/new`;
- call `runner.start_new_session(...)` with adapter route metadata;
- call `runner.resume_session(...)` for `/resume`;
- bind `SessionRouteBinding` to `runner.interaction_router` after a switch;
- translate unavailable/missing-session errors into adapter-local messages.

Deletion test: deleting a shared session switch interface today would force the start/resume route-binding rules back into TUI, generic text channels, and Telegram.

## This Loop Scope

Deepen `demiurge/runtime/session_commands.py` so it owns session start/resume route-binding rules.

This is behaviour-preserving. It does not change session storage, session IDs, route tokens, `/new` or `/resume` visible messages, TUI history restoration, channel command text, Agent Core layout, Agent Slot manifests, Package recipes, or authored SDK methods.

## Planned Code Changes

- Add a small `SessionSwitchResult` result type in `demiurge/runtime/session_commands.py`.
- Add `start_bound_session(...)` for `/new`.
- Add `resume_bound_session(...)` for `/resume`.
- Update `TuiInteractionBridge`, `TextChannelBridgeBase`, and `TelegramInteractionBridge` to call these runtime functions.
- Add focused runtime tests for:
  - starting a bound session prepares the live core and binds the route;
  - unavailable start returns a stable error result;
  - resuming a bound session binds the route;
  - missing session errors become a non-throwing result.

## Out Of Scope

- Changing session persistence.
- Changing session list rendering.
- Changing TUI history snapshot behavior after resume/new.
- Changing channel command names or text.
- Changing route token internals.
- Copying Hermes implementation code.

## Expected Benefits

- Session switch behavior becomes one deeper runtime module interface.
- Locality improves: route-binding bugs concentrate in `runtime/session_commands.py`.
- Leverage improves: TUI, generic text channels, and Telegram share one start/resume rule.
- Adapters keep adapter responsibilities: route metadata, visible notices, and platform delivery.
