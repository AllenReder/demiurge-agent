## Reference Comparison

### Hermes reference

Hermes derives slash command surfaces from one command registry:

- `hermes_cli/commands.py` owns `COMMAND_REGISTRY`, aliases, gateway availability, lookup helpers, autocomplete, and `gateway_help_lines()`.
- `gateway/slash_commands.py` calls `gateway_help_lines()` for platform help instead of carrying a separate help list.
- Telegram BotCommands, Slack command mapping, autocomplete, and gateway help all derive from the same registry.

The transferable lesson is not Hermes's exact registry fields. Demiurge should make its existing `demiurge/slash.py` catalog deep enough that adapters ask it for surface names and help text instead of carrying local derived views.

### Demiurge current state

This branch now has:

- `ChannelCommandRuntime` for command classification and dispatch.
- `Command` metadata in `demiurge/slash.py`.
- `SessionCommand` read-model extraction in `demiurge/runtime/session_commands.py`.

The remaining shallow command catalog cluster is:

- `demiurge/slash.py` owns `SLASH_COMMANDS`, but only exposes raw specs and Telegram BotCommand filtering.
- `TextChannelBridgeBase` still hardcodes `TEXT_CHANNEL_COMMAND_NAMES`.
- `TextChannelBridgeBase._command_help(...)` still hardcodes text-channel help.
- `TuiInteractionBridge._help(...)` loops over all `SLASH_COMMANDS`, so its help surface can mention commands that are not TUI commands.
- `TelegramInteractionBridge` derives help and command names from `specs_for_surface("telegram")`, but repeats formatting locally.

Deletion test: deleting deeper catalog helpers would force command names and help-text derivation back into TUI, Text, and Telegram adapters.

## This Loop Scope

Deepen `demiurge/slash.py` as the command catalog read-model module.

This is behaviour-preserving except for aligning help output with declared command surfaces. It does not change command handler side effects, Agent Core layout, Agent Slot manifests, Package recipes, authored SDK methods, command parsing, `/ask` pass-through semantics, session/runtime storage, or Telegram transport behavior.

## Planned Code Changes

- Extend `SlashCommandSpec.surfaces` metadata to include a generic `text` surface for the existing text-channel command set.
- Add catalog helpers in `demiurge/slash.py`:
  - `command_names_for_surface(surface, specs=SLASH_COMMANDS)`;
  - `help_text_for_surface(surface, specs=SLASH_COMMANDS, extra_lines=(), footer_lines=())`.
- Keep `telegram_command_specs(...)` as the BotCommand-specific adapter view, but build command names through the new helper where appropriate.
- Update `TextChannelBridgeBase`:
  - replace `TEXT_CHANNEL_COMMAND_NAMES` with `command_names_for_surface("text")`;
  - replace hardcoded help text with `help_text_for_surface("text", extra_lines=(... /ask ...))`.
- Update `TelegramInteractionBridge`:
  - use `command_names_for_surface("telegram")`;
  - use `help_text_for_surface("telegram")` for `/help`.
- Update `TuiInteractionBridge`:
  - use `help_text_for_surface("tui", footer_lines=(...))`;
  - stop importing raw `SLASH_COMMANDS` for help rendering.
- Add focused tests for command catalog helpers:
  - text command names match the previous text command set;
  - TUI help excludes Telegram-only `/stop` and `/queue`;
  - text help can include `/ask <prompt>` without putting `/ask` into the dispatch set;
  - Telegram BotCommands still exclude non-Telegram surfaces and invalid names.

## Out Of Scope

- Moving command handler execution out of adapters.
- Moving `/status`, `/tools`, `/skills`, `/skill`, `/subagents`, `/new`, `/busy`, or `/resume` side effects.
- Adding aliases or a Hermes-style command worker.
- Changing `/ask` parsing or dispatch semantics.
- Changing Telegram command registration limits.
- Changing Agent Core / Agent Slot / Package authored surface.
- Copying Hermes implementation code.

## Expected Benefits

- Command vocabulary and derived help/name views have one deeper module interface.
- Locality improves: surface membership, help text grouping, and command-name sets live in `slash.py`.
- Leverage improves: TUI, Text, Telegram, and future dashboard/Web UI surfaces reuse one catalog.
- Tests target the catalog interface directly instead of adapter-private hardcoded lists.
