# 08:23:25 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 prompt delivery 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema。

## Hermes 对比

- Hermes 的 clarify gateway primitive 把 register、wait、resolve、session cleanup 放在独立 module；平台 adapter 负责把同一个 prompt interface 渲染为平台 UI，并把 button/text response 回传到 shared primitive。
- Hermes Telegram adapter 的 clarify 按钮只保留 Telegram auth、message id、inline keyboard 等 adapter 知识；choice text 解析和等待者唤醒不散落在每个 send path。
- 当前 Demiurge 已有 `demiurge.runtime.prompts`，并且已经集中 choice parsing、callback data、button rows 和 `PromptChoiceRuntime`。不能重复实现那一层。
- 当前剩余浅模块是 prompt delivery orchestration：
  - `TextChannelBridgeBase.prompt_user()` 自己 remember choices、读取 `source` / `reply_to`、格式化 prompt 文本、发送文本。
  - `TelegramInteractionBridge.prompt_user()` 复制同样逻辑，再添加 Telegram `reply_markup`。
  - Text 与 Telegram inbound 都需要从 pending choice 中归一化 answer，但这仍由 adapter 拼接 `InteractionInbound`。

## 设计

在 `demiurge.runtime.prompts` 中深化一个 prompt delivery module：

- 新增 `PromptDelivery`：包含 `source`、`text`、`reply_to`、`metadata`、`choices`，是 adapter 发送 prompt 所需的最小 interface。
- 新增 `PromptDeliveryRuntime`：
  - `prepare(prompt)`：集中记忆 prompt choices、抽取 `source` / `reply_to`、生成 numbered fallback text。
  - `resolve_inbound(inbound)`：集中把 pending choice reply 映射回 canonical choice text，并返回新的 `InteractionInbound`。
  - `consume_callback_data(conversation_key, data)`：集中 Telegram-style callback choice 解析。
  - `pending_choices(...)`：测试和 adapter auth regression 使用的只读检查 interface。
- Text channel 只调用 `prepare()` 并把 `PromptDelivery` 交给 `_send_text()`。
- Telegram channel 只调用 `prepare()`，并用 `PromptDelivery.choices` 组装 `reply_markup`；callback auth 和 Telegram message id 仍属于 adapter。
- 删除 adapter 内重复的 pending choice field、prompt text formatting、inbound reconstruction logic。

## 优势

- locality：prompt delivery bug 集中在一个 module，而不是 Text/Telegram 双写。
- leverage：两个 adapter 和 runtime tests 共用同一 interface，后续 Discord/Matrix/WhatsApp channel 不再复制。
- depth：adapter interface 从 “知道 choice lifecycle” 收缩为 “发送 prepared prompt”。
- testability：新增 runtime tests 覆盖 prepare/resolve/callback；channel tests 只验证 adapter wiring。

## Implementation steps

1. 扩展 `demiurge/runtime/prompts.py`：添加 `PromptDelivery` 和 `PromptDeliveryRuntime`。
2. 扩展 `tests/runtime/test_prompts.py`：覆盖 prepare、missing source、inbound resolution、callback resolution。
3. 更新 `demiurge/channels/base.py`：用 `PromptDeliveryRuntime` 替代 `_pending_choices` 和 `_consume_inbound_pending_choice`。
4. 更新 `demiurge/channels/telegram/bridge.py`：用 `PromptDeliveryRuntime` 替代 `_pending_choices`，把 text reply resolution 从 `normalize_update()` 移到 authorized `handle_update()` 后。
5. 更新 `tests/channels/telegram/test_channels.py` 里私有字段断言和 prompt text reply 行为测试。
6. 运行验证：
   - `uv run pytest tests/runtime/test_prompts.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py -q`
   - `uv run python -m compileall demiurge/runtime/prompts.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/runtime/test_prompts.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py`
   - `git diff --check -- demiurge/runtime/prompts.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/runtime/test_prompts.py tests/channels/telegram/test_channels.py tests/channels/test_text_bridge.py goal/08-23-25.md`

## Non-goals

- 不实现 Hermes-style blocking clarify store。
- 不改变 approval request/decision lifecycle。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
