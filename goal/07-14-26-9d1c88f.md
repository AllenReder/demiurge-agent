# 07:14:26 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 approval interaction 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema，不改变 host-owned approval security policy。

## Hermes 对比

- Hermes 的 gateway approval 体系把 blocking approval、Telegram inline buttons、`/approve` / `/deny` fallback 和 session cleanup 作为独立控制面处理。平台 adapter 负责传输，approval primitive 负责 action 语义。
- 当前 Demiurge 已有 `ApprovalRequest`、`ApprovalDecision` 和 `ApprovalRuntime`，安全策略集中在 host 层；但用户交互语义仍散在 adapter：
  - TUI 在 `parse_approval_response()` 中维护 `allow` / `always_allow_for_session` / `deny` 的文本别名。
  - Telegram 在 bridge 内维护 callback 格式、button rows、action -> decision 映射、callback answer 文案、resolved approval 文案。
  - `ApprovalRuntime` 只验证 decision value，不知道用户交互 action 的 canonical surface。
- 这还不是完整 Hermes-style blocking approval queue/store，但可以先把纯 in-process action semantics 收拢，为后续 pending approval store 和 `/approve` fallback 做准备。

## 设计

新增 `demiurge.runtime.approvals`：

- `ApprovalCallback`：保存从 callback data 中解析出的 `approval_id` 和 action。
- `parse_approval_response(text)`：统一 TUI/free-text approval 输入。
- `approval_decision_for_action(action)`：统一 `allow` / `session` / `deny` 到 `ApprovalDecision` 的映射。
- `approval_callback_data(approval_id, action)` 和 `parse_approval_callback_data(data)`：统一 Telegram callback namespace。
- `approval_button_rows(approval_id)`：统一 Telegram inline button rows。
- `approval_callback_answer(decision)`：统一 callback acknowledgement。
- `approval_resolution(action)`：统一 resolved message 的 title/detail。
- `format_approval_request_text(request)` 和 `format_resolved_approval_text(request, title, detail)`：把 request/resolved message 文案从 Telegram adapter 中移出。

Adapter 接线：

- `TuiInteractionBridge.reply_approval()` 改用 shared `parse_approval_response()`，删除本地 implementation。
- `TelegramInteractionBridge` 改用 shared callback parse、button rows、decision mapping、ack 文案、resolved title/detail、approval request/resolved text。
- Telegram 继续持有 pending future、message id 和 transport edit/send；这属于 adapter transport seam，本轮不抽 pending store。

## 优势

- locality：approval action alias、callback namespace、button label、resolved copy 的 bug 集中在一个 module。
- leverage：TUI 和 Telegram 同时复用 action semantics。
- depth：adapter interface 缩小为 “发送 approval request / 等待 future / 回调 shared resolver / 编辑消息”。
- testability：新增 `tests/runtime/test_approvals.py` 直接覆盖 shared interface，现有 TUI/Telegram tests 继续覆盖 wiring。

## Implementation steps

1. 新增 `demiurge/runtime/approvals.py` 和 `tests/runtime/test_approvals.py`。
2. 更新 `demiurge/ui_gateway/bridge.py` 使用 shared `parse_approval_response()`。
3. 更新 `demiurge/channels/telegram/bridge.py` 使用 shared approval interaction helpers，并删除本地 `_approval_reply_markup` / `_approval_text` / `_resolved_approval_text` 的 implementation。
4. 更新 Telegram tests 中对 callback markup 的断言，继续验证 bridge wiring。
5. 运行验证：
   - `uv run pytest tests/runtime/test_approvals.py tests/channels/test_ui_gateway.py tests/channels/telegram/test_channels.py -q`
   - `uv run python -m compileall demiurge/runtime/approvals.py demiurge/ui_gateway/bridge.py demiurge/channels/telegram/bridge.py tests/runtime/test_approvals.py tests/channels/test_ui_gateway.py tests/channels/telegram/test_channels.py`
   - `git diff --check -- demiurge/runtime/approvals.py demiurge/ui_gateway/bridge.py demiurge/channels/telegram/bridge.py tests/runtime/test_approvals.py tests/channels/test_ui_gateway.py tests/channels/telegram/test_channels.py goal/07-14-26.md`

## Non-goals

- 不改变 `ApprovalRuntime` 的 security policy、session allowlist 或 event emission。
- 不实现完整 pending approval queue/store。
- 不实现 Telegram `/approve` / `/deny` text fallback。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
