## Reference Comparison

### Hermes reference

Hermes keeps command dispatch out of individual platform senders:

- `gateway/slash_commands.py` extracts command handlers from the gateway runner so the command module has an explicit home.
- `gateway/platforms/base.py` handles active-session command bypass centrally before ordinary platform messages become pending follow-up messages.
- Hermes command tests such as `tests/gateway/test_command_bypass_active_session.py` treat command classification as a gateway invariant, not a Telegram-only branch.

The transferable lesson is not Hermes's full command catalogue. Demiurge should first name command classification and dispatch for external channels so each adapter stops repeating slash parse, `/ask` rewrite, allowlist, handler lookup, and unknown-command behaviour.

### Demiurge current state

This branch now has:

- `CompletionInbox` for background completion claim and ack.
- `InboundQueueRuntime` for inbound queue selection and completion merge.
- `ConversationIngressState` for route memory and completion-route construction.
- `ConversationTurnController` for active-task, queue-drain, cancel, and completion-enqueue lifecycle rules.

The remaining shallow cluster is channel command dispatch:

- `TextChannelBridgeBase.handle_inbound(...)` and `TelegramInteractionBridge.handle_inbound(...)` both parse slash commands and implement the same `/ask` rewrite.
- Both adapters compare the parsed command against a command-name set, dispatch to a local handler map, and render unknown command text.
- The command handler maps remain adapter-local, but the classification and dispatch rules are the same module concept.

Deletion test: deleting either adapter's command classification would force the same parse/rewrite/allowlist/unknown logic to reappear in the other adapter. Two adapters justify a real seam.

## This Loop Scope

Extract a `ChannelCommandRuntime` module for external channel command classification and dispatch.

This is behaviour-preserving. It changes internal module placement and adapter call sites only; it does not change Agent Core layout, Agent Slot manifests, Package recipes, authored SDK methods, command names, command output text, busy mode meanings, approval semantics, delivery rendering, task completion schema, TUI command handling, or Telegram polling behaviour.

## Planned Code Changes

- Add `demiurge/channels/commands.py` with:
  - `ChannelCommandOutcome`;
  - `ChannelCommandRuntime`.
- Move shared command classification into `ChannelCommandRuntime`:
  - parse slash command text;
  - rewrite `/ask <prompt>` into an ordinary inbound prompt;
  - test command availability from an injected command-name set;
  - call an injected handler map for available commands;
  - call injected notice text for unknown or unavailable commands;
  - return a small outcome so adapters know whether to continue as a normal turn.
- Update `TextChannelBridgeBase`:
  - instantiate a command runtime with the current text-channel command-name set;
  - delegate command classification and handler dispatch to the runtime;
  - keep text-channel command handlers and text formatting local.
- Update `TelegramInteractionBridge`:
  - instantiate a command runtime with `specs_for_surface("telegram")`;
  - delegate command classification and handler dispatch to the runtime;
  - keep Telegram command handlers, command registration, mention normalization, and text formatting local.
- Add focused tests in `tests/channels/test_commands.py`:
  - no slash command passes through unchanged;
  - `/ask <prompt>` rewrites to ordinary inbound text and is not handled;
  - available command calls its handler and stops the turn;
  - available command without a handler sends unavailable notice and stops the turn;
  - unavailable command sends unknown notice and stops the turn.

## Expected Benefits

- Channel command dispatch becomes one deep module interface instead of two copied adapter branches.
- Locality improves: command parse, `/ask` rewrite, allowlist, handler lookup, and unknown/unavailable behaviour live together.
- Leverage improves: text channels and Telegram reuse the same command dispatch module.
- Tests can target the command interface directly, matching the codebase-design principle that the interface is the test surface.
