# 07:52:44 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 conversation state registry 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema，不改变 turn/session durable state machine。

## Hermes 对比

- Hermes 的 gateway/session 文档把 `SessionSource`、session key、gateway ownership 和 delivery routing 作为 session lifecycle 的一等对象，而不是让每个 adapter 分别维护 routing glue。
- 当前 Demiurge 已有 `ConversationIngressState` / `SessionRouteBinding` / `ConversationTurnController`，但 Text 和 Telegram adapter 仍重复维护：
  - `conversation_key -> ConversationIngressState` cache。
  - `session_id -> state` 查找，用于 background completion delivery。
  - 从 runtime runner 中取 `task_worker` 并订阅 completion callbacks。
- 更重要的是，当前 bridge 只保存一个 `_task_worker` / `_task_unsubscribe`。若 `runtime_factory` 为不同 conversation 返回不同 runner，后创建的 conversation 会替换订阅，早期 conversation 的 completion callback 不再投递到 bridge。

## 设计

新增 `demiurge.runtime.conversation_states`：

- `ConversationStateStore`：
  - 初始化参数：`state_factory`、`on_task_completion`。
  - `state_for_key(conversation_key)`：按 conversation key 懒创建并缓存 `ConversationIngressState`。
  - `state_for_session(session_id)`：统一通过 cached state 的 live `session_id` 查找 completion route owner。
  - 自动订阅每个 distinct `RuntimeTaskWorker`，而不是只保留 latest worker。
  - `close()`：统一释放所有 task worker subscriptions。
- 该 module 是 adapter-facing runtime seam；它不拥有 session persistence，不替代 `SessionRuntime`、`ConversationTurnController` 或 durable runtime work。

Adapter 接线：

- `TextChannelBridgeBase` 用 `ConversationStateStore` 替代本地 `_conversation_state()` cache 和 `_subscribe_task_worker()` implementation。
- `TelegramInteractionBridge` 同样改用 shared store，并保留 `_conversations` dict 作为 tests/adapter-local state inspection 的兼容属性。
- Text/Telegram `CompletionDeliveryRuntime` 不再传 bridge-level latest `_task_worker`，而是让 `ConversationIngressState` 按当前 state 的 runner worker claim completion event。

## 优势

- locality：conversation cache、session lookup、task-worker subscription policy 集中到一个 module。
- leverage：Text 和 Telegram 同时复用；未来 Mattermost/Discord style adapters 不需要复制该 policy。
- depth：adapter interface 缩小为 “如何创建一个 state” 和 “收到 completion event 后如何 enqueue”。
- testability：新增 runtime-level tests 直接覆盖 state cache、multi-worker subscriptions、session lookup 和 unsubscribe，而无需通过 Telegram polling 细节验证。

## Non-goals

- 不改变 TUI 单 session ingress lifecycle。
- 不改变 session persistence、conversation key 生成规则或 `/sessions` / `/resume` 行为。
- 不改变 Text/Telegram user-visible busy/queue/cancel copy。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
