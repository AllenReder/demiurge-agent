# 08:11:21 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的 text outbound delivery runtime 深化。保留 Agent Core、Agent Slot、Package 能力包概念不变；不修改 authored surface，不修改 package schema，不改变 session/turn durable state machine。

## Hermes 对比

- Hermes 的 gateway delivery path 把 delivery routing、oversize handling、local audit save 和 platform adapter send 分开；平台 adapter 负责实际平台发送能力，而不是重复完整 delivery orchestration。
- 当前 Demiurge 已有 `demiurge.runtime.outbound_delivery.text_delivery_steps()`，但 step dispatch 仍重复在两个 adapter：
  - `TextChannelBridgeBase.deliver()` 遍历 `text_delivery_steps()`，按 `tool_call` / `tool_results` / `delivery` / `prompt` 分派，并在 `finally` 中 `outbound.mark_delivered()`。
  - `TelegramInteractionBridge.deliver()` 复制同一调度循环和同一 delivered invariant。
- 这使 `outbound_delivery` 仍然浅：它只规划 steps，关键的 delivery lifecycle invariant 仍留在每个 adapter。

## 设计

扩展 `demiurge.runtime.outbound_delivery`：

- 新增 `TextOutboundDeliveryRuntime`：
  - 初始化参数：`deliver_tool_call`、`deliver_tool_results`、`deliver_delivery`、`prompt_user`。
  - `deliver(outbound)`：统一执行 `text_delivery_steps(outbound)`，分派 step，并保证 `outbound.mark_delivered()` 在 finally 中执行。
- 该 module 是 text-like adapter outbound seam；它不拥有平台发送、Telegram edit/fallback、media block rendering 或 TUI batch display。

Adapter 接线：

- `TextChannelBridgeBase.deliver()` 改成调用 `TextOutboundDeliveryRuntime(...).deliver(outbound)`。
- `TelegramInteractionBridge.deliver()` 改成调用同一 runtime。
- TUI 保持使用 `ui_delivery_steps()`，因为它按 UI event batch 展示 tool calls/deliveries，不是 text-like outbound adapter。

## 优势

- locality：text outbound step dispatch 和 delivered invariant 集中在一个 module。
- leverage：Text 和 Telegram 同时复用，后续 text-like adapters 可直接接入。
- depth：adapter interface 缩小为 “发送这类 step”，不再知道 delivery lifecycle orchestration。
- testability：新增 runtime-level tests 覆盖 step dispatch 顺序与 exception path 的 `mark_delivered()` invariant；existing channel tests 覆盖 adapter wiring。

## Implementation steps

1. 扩展 `demiurge/runtime/outbound_delivery.py`，新增 `TextOutboundDeliveryRuntime` 和 callable protocols。
2. 扩展 `tests/runtime/test_outbound_delivery.py`，覆盖 text runtime dispatch order 和 exception still marks delivered。
3. 更新 `demiurge/channels/base.py` 使用 `TextOutboundDeliveryRuntime`。
4. 更新 `demiurge/channels/telegram/bridge.py` 使用 `TextOutboundDeliveryRuntime`。
5. 运行验证：
   - `uv run pytest tests/runtime/test_outbound_delivery.py tests/channels/test_text_bridge.py tests/channels/telegram/test_channels.py -q`
   - `uv run python -m compileall demiurge/runtime/outbound_delivery.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/runtime/test_outbound_delivery.py`
   - `git diff --check -- demiurge/runtime/outbound_delivery.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/runtime/test_outbound_delivery.py goal/08-11-21.md`

## Non-goals

- 不改变 `text_delivery_steps()` 或 `ui_delivery_steps()` 的 step planning semantics。
- 不改变 Telegram rich media delivery、tool message edit/fallback、reply threading 或 message splitting。
- 不改变 TUI delivery batching。
- 不改变 Agent Core / Agent Slot / Package 能力包概念或文件格式。
