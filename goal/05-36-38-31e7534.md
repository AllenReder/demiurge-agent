## Reference Comparison

### Hermes reference

Hermes's TUI/gateway path keeps terminal presentation concerns in focused client/domain files rather than letting every adapter carry its own truncation and display rules. Hermes tools also use explicit output-limit modules for result shaping, which gives one place to change user-facing truncation policy.

The transferable lesson is not Hermes's exact UI implementation. Demiurge should centralize pure text read-model rules that many runtime and adapter modules already share: table rendering, key-value rendering, safe JSON coercion, and preview shortening.

### Demiurge current state

This branch now has deeper read-model modules for:

- slash command catalog;
- session list/resume;
- runtime status;
- tool display content.

The remaining shallow formatting cluster is:

- `demiurge/runtime/session_commands.py`: local `format_table(...)` and `shorten_text(...)`;
- `demiurge/runtime/status_commands.py`: local `format_table(...)` and `shorten_text(...)`;
- `demiurge/runtime/delegation.py`: local table renderer;
- `demiurge/runtime/tool_display.py`: local `shorten_tool_text(...)` and `json_safe(...)`;
- `demiurge/ui_gateway/bridge.py`: local `_format_table(...)`, `_format_key_values(...)`, `_json_safe(...)`, and `shorten_text(...)`.

Deletion test: deleting one shared formatting module today would force the same width, truncation, and JSON-safety rules to remain scattered across runtime read-models and TUI adapter helpers.

## Expected Benefits

- Text formatting becomes one deeper module interface.
- Locality improves: width, truncation, and JSON-safety changes happen in one place.
- Leverage improves: session/status/delegation/tool/TUI read-models share the same tested implementation.
- Adapter modules stop owning generic table rules and retain only delivery-specific behavior.
