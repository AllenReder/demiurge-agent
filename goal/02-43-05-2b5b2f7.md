## Reference Comparison

### Hermes reference

Hermes keeps delegated subagent lifecycle in named Modules:

- `tools/delegate_tool.py` owns child construction, tool restrictions, depth/concurrency limits, active child records, and result shaping.
- `tools/async_delegation.py` owns detached child dispatch and completion re-entry.
- `tools/process_registry.py` shows the same pattern for background work: a named runtime tracks status, logs, wait/cancel, checkpointing, and completion notifications.
- `docs/observability/README.md` treats subagent start/stop as a first-class lifecycle event family with parent/child correlation IDs.

The transferable lesson is not Hermes code shape. Demiurge already has `RuntimeTaskWorker` for durable background work. The missing Module is the child-agent lifecycle that sits above that worker and below Agent Slot SDK calls.

### Demiurge current state

This branch now has:

- `TurnEngine` for model/tool loop orchestration.
- `SlotPipelineRuntime` for input/output phase scheduling.
- `SlotContextRuntime` for authored `ctx.*` construction.
- `TurnIO` for model-step transcript and interaction-item assembly.
- `ModuleDeliveryRuntime` for authored delivery request application.

The remaining runner-owned child-agent cluster includes:

- `_resolve_child_agent_slots(...)` and slot request metadata helpers;
- `_resolve_child_agent_tools(...)`, MCP-aware preparation, and child tool policy construction;
- `_run_child_agent(...)` child runner construction, session metadata, child slot/tool metadata, result projection, and needs-user detection;
- `_spawn_child_agent(...)` RuntimeTaskWorker submission, spawn lifecycle events, completion metadata, and failure projection;
- `_delegate_task(...)` model-facing delegation validation, depth/concurrency limits, notify policy, context-mode handling, and child spawn.

Deletion test: deleting this runner block would force the same slot/tool/lifecycle rules to reappear in `SlotContextRuntime`, `ToolRuntime`, and runtime task handling. It is a deep Module trapped in the runner.

## This Loop Scope

Extract a `ChildAgentRuntime` Module that owns child-agent slot/tool resolution, synchronous child runs, spawned child tasks, and `delegate_task` request handling.

This is behavior-preserving. It changes internal Module placement and host Adapter shape only; it does not change Agent Core layout, Agent Slot manifests, Package recipes, authored SDK methods, task tool semantics, channel behavior, bridge behavior, TUI behavior, or outbox behavior.

## Planned Code Changes

- Add `demiurge/runtime/child_agents.py`.
- Move child-agent constants and data classes into that Module:
  - `ChildSlotRequest`, `ChildToolRequest`;
  - `ResolvedPhaseSlots`;
  - `ResolvedChildAgentSlots`;
  - `ResolvedChildAgentTools`;
  - child-agent default input/output slot IDs, `"all"`, and `"none"` sentinels.
- Add a `ChildAgentHost` Protocol and `RunnerChildAgentHost` Adapter.
- Add `ChildAgentRuntime` methods:
  - `resolve_slots(...)`;
  - `resolve_slots_for_core_id(...)`;
  - `resolve_tools(...)`;
  - `resolve_tools_for_core_id_prepared(...)`;
  - `requested_tools_for_core_id(...)`;
  - `run_child(...)`;
  - `spawn_child(...)`;
  - `handle_delegate_task(...)`;
  - `delegation_context(...)`.
- Instantiate `self.child_agents = ChildAgentRuntime(RunnerChildAgentHost(self))` in runner.
- Update runner:
  - `run_turn(...)` imports `ResolvedPhaseSlots` from `child_agents`;
  - `_run_child_agent(...)` delegates to `self.child_agents.run_child(...)`;
  - `_spawn_child_agent(...)` delegates to `self.child_agents.spawn_child(...)`;
  - `_delegate_task(...)` delegates to `self.child_agents.handle_delegate_task(...)`;
  - `_delegation_context(...)` delegates to `self.child_agents.delegation_context(...)`;
  - remove runner-local child slot/tool helper methods and constants that become unused.
- Update `SlotContextRuntime` imports to use child-agent request types only where useful.
- Add focused tests in `tests/runtime/test_child_agents.py`:
  - child slot defaults and `"all"` pipeline selection;
  - invalid child slot requests;
  - tool selection for `"all"`, `"none"`, explicit allowlist, duplicate/empty/missing IDs;
  - delegate_task context-mode/depth/notify-policy validation where it can be tested directly.
- Keep these concerns host-owned for this loop:
  - generic `task_status`, `task_control`, and `yield_until`;
  - generic tool execution outside `delegate_task`;
  - TurnEngine and SlotPipelineRuntime behavior;
  - RuntimeTaskWorker internals.

## Expected Benefits

- `SessionTurnStepRunner` loses child-agent lifecycle responsibility.
- `SlotContextRuntime` calls a smaller runner-host seam for `ctx.agents.run/spawn`.
- Child slot/tool policy and spawn lifecycle become directly testable through one Module Interface.
- Future runtime-task control and interaction route redesign get a narrower, named Module to work against.
- Hermes parity improves conceptually: subagent lifecycle becomes an explicit Module with parent/child correlation metadata rather than incidental runner glue.
