# 06:43:32 Hermes-informed redesign plan

## Scope

本轮推进 “demiurge 2.0” 设计中的外部 channel control surface 深化。保留 Agent Core、Agent Slot、Package 能力包这三个概念不变；本轮不改 authored surface，也不改 package schema。

## Hermes 对比

- Hermes 的 gateway 把 session lifecycle 和 slash command 规则放在 gateway 层，平台 adapter 主要负责把平台事件转成统一消息、再把结果发回平台。
- 当前 Demiurge 已经通过前几轮把 session/status/history/outbound/completion 的部分规则抽到 runtime module，但外部 channel command 仍然重复在 `TextChannelBridgeBase` 和 `TelegramInteractionBridge` 中。
- 这种重复导致 `/new`、`/stop`、`/queue`、`/busy`、`/sessions`、`/resume`、`/tools`、`/skills`、`/skill` 的运行时规则需要在不同 adapter 中同步维护。Telegram 只应保留平台能力差异，例如 command mention 过滤、rich message、inline callback、approval message edit、access status 附加行。

## 设计

新增或深化 `demiurge.channels.commands`，让它从浅的 parse/dispatch module 变成外部 channel command executor：

- `ChannelCommandRuntime` 继续拥有 slash command parse、`/ask` rewrite、unknown/unavailable command 处理。
- 新增 deep module `ChannelCommandExecutor`，拥有外部 channel 共享命令实现。
- Adapter 只提供小 interface：
  - `send_text(inbound, text)`：按平台发送一段 command 文本。
  - `run_inbound(state, inbound)`：启动或排队普通 inbound。
  - `cancel_active(state)`：取消当前 turn；Telegram 可在取消前清理 pending approval。
  - `status_extra_lines(inbound)`：Telegram 注入 allowlist/access 状态，普通 text channel 不需要。
- `TextChannelBridgeBase` 和 `TelegramInteractionBridge` 删除重复 command handler 方法，只保留 adapter-specific hooks。

## 优势

- locality：外部 channel command 行为集中在一个 module；修 `/queue`、`/busy`、`/resume` 不再跨 adapter。
- leverage：所有 TextChannelBridgeBase 子类和 Telegram 通过同一 command interface 受益。
- tests：新增 `tests/channels/test_commands.py` 覆盖 shared command behaviour；Telegram/Text 现有 tests 继续覆盖 adapter wiring。
- 破坏性重构准备：后续可以继续把 prompt/approval lifecycle 和 route context 深化，而不用先处理 command duplication。

## Implementation steps

1. 在 `demiurge/channels/commands.py` 添加 `ChannelCommandExecutor`、command config/callback 类型和 shared command handlers。
2. 改 `demiurge/channels/base.py`：
   - 初始化 `ChannelCommandExecutor(surface="text")`。
   - `_handle_command` 使用 executor handlers。
   - 删除重复 command handler 方法和不再需要的 imports。
3. 改 `demiurge/channels/telegram/bridge.py`：
   - 初始化 `ChannelCommandExecutor(surface="telegram")`。
   - 提供 Telegram status extra lines 和 cancel-active hook。
   - 删除重复 command handler 方法和不再需要的 imports。
4. 更新 `tests/channels/test_commands.py`，用 fake runtime/state 验证 shared command executor 的 `/busy`、`/queue`、`/tools` 或 `/skill` 行为。
5. 跑聚焦验证：
   - `uv run pytest tests/channels/test_commands.py tests/channels/telegram/test_channels.py tests/channels/test_ui_gateway.py -q`
   - `uv run python -m compileall demiurge/channels/commands.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/channels/test_commands.py`
   - `git diff --check -- demiurge/channels/commands.py demiurge/channels/base.py demiurge/channels/telegram/bridge.py tests/channels/test_commands.py goal/06-43-32.md`

## Non-goals

- 不改 TUI command set。
- 不改 Agent Core、Agent Slot、Package manifest。
- 不把 Telegram approval lifecycle 抽出来；它已从旧状态演进为可交互实现，本轮只避免 command 规则继续重复。
